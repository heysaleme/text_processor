# Text Processor

CLI-инструмент на Go для обработки текстовых файлов по набору правил: преобразование встроенных команд, нормализация пунктуации, работа с кавычками и коррекция артиклей `a/an`.

Проект написан только на стандартной библиотеке Go и запускается в формате:

```bash
go run . input.txt output.txt
```

## Что делает

- применяет команды `(up)`, `(low)`, `(cap)`, `(hex)`, `(bin)`
- поддерживает формы `(up, N)`, `(low, N)`, `(cap, N)` для нескольких предыдущих слов
- нормализует пробелы вокруг `.,!?:;`
- сохраняет последовательности вроде `...` и `!?`
- корректно собирает одинарные кавычки и распространённые сокращения
- исправляет артикли `a/an` по следующему слову

## Архитектура

Проект оставлен в одном пакете `main`, но разделён по ответственности:

- [processor.go](/Users/amira/Desktop/text_processor/processor.go) — общий pipeline обработки текста

- [articles.go](/Users/amira/Desktop/text_processor/articles.go) — логика коррекции `a/an`
- [reconstruct.go](/Users/amira/Desktop/text_processor/reconstruct.go) — восстановление итогового текста, пунктуация, кавычки, сокращения
- [helpers.go](/Users/amira/Desktop/text_processor/helpers.go) — общие вспомогательные функции

Такое разделение делает код проще для чтения, проверки и точечного тестирования без искусственного усложнения структуры.

## Pipeline обработки

1. Входной текст токенизируется на слова, команды, знаки препинания, кавычки и переводы строк.
2. Артикли `a/an` корректируются с учётом следующего релевантного слова.
3. Команды применяются к предыдущим словам, включая формы с количеством и сложные вложенные конструкции.
4. Итоговый текст собирается обратно с нормализацией пробелов, пунктуации и кавычек.

## Сложные кейсы

Проект рассчитан не только на базовые случаи, но и на более тяжёлые входы:

- несколько команд подряд: `ab (up) (hex) (low)`
- вложенные команды: `(Up (low))`, `(LOW(low))`, `(L(low)O(low)W(low))`
- команды с количеством: `(cap, 6)`, `(low, 3)`, `(up, 2)`
- комбинации числовых преобразований с большими значениями через `math/big`
- смешанная пунктуация: `...`, `!?`, `!!`
- разорванные кавычки и сокращения: `I don ' t know`, `it ' s`
- многострочный ввод с сохранением структуры строк

Именно эти кейсы чаще всего ломают простые реализации, поэтому логика разделена на отдельные этапы: токенизация, трансформации, коррекция артиклей и реконструкция текста.

## Дисклеймер по `h`

В проекте правило для `a/an` реализовано с грамматическими исключениями для слов с буквой `h`.

Это означает:

- `an hour`, `an honest`, `an heir`
- но `a house`, `a hospital`, `a history book`

Такой подход ближе к реальному английскому языку, но может не совпадать с упрощённой формулировкой некоторых учебных заданий, где требуется заменять `a` на `an` перед любым словом, начинающимся с `h`.

## Пример

Вход:

```text
harold wilson (cap, 2) : ' I am a optimist ,but a optimist who carries a raincoat . '
it (cap) was the best of times, it was the worst of times (up) , it was the age of foolishness (cap, 6) .
42 (hex) and 10 (bin)
I don ' t know what to say ! !
```

Выход:

```text
Harold Wilson: 'I am an optimist, but an optimist who carries a raincoat.'
It was the best of times, it was the worst of TIMES, It Was The Age Of Foolishness.
66 and 2
I don't know what to say!!
```

