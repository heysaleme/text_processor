# 📘 go-reloaded

**go-reloaded** — это утилита на Go для обработки текста, которая исправляет пробелы и пунктуацию, корректирует артикли **a/an** и применяет встроенные команды форматирования прямо внутри текста.

---

## ✨ Возможности

### 🔹 Нормализация пробелов и пунктуации
- Удаляет лишние пробелы перед знаками препинания
- Добавляет пробел после запятых, точек, двоеточий и точек с запятой при необходимости
- Обрабатывает многоточия (`...`) и множественные знаки вопроса/восклицания (`??`, `!!`)

### 🔹 Автоматическое исправление артиклей
- `a apple` → `an apple`
- `an hat` → `a hat`
- Учитывает исключения: `hour`, `honest`, `heir`, `university`, `unicorn` и др.

### 🔹 Поддержка встроенных команд форматирования
Скобочные команды применяются к предыдущим словам:

| Команда        | Действие                                 | Пример                     |
|---------------|------------------------------------------|----------------------------|
| `(up)`        | Верхний регистр                          | `hello (up)` → `HELLO`     |
| `(low)`       | Нижний регистр                           | `HELLO (low)` → `hello`    |
| `(cap)`       | Capitalize (первая буква заглавная)      | `word (cap)` → `Word`      |
| `(hex)`       | Перевод из HEX в десятичную систему      | `1F (hex)` → `31`          |
| `(bin)`       | Перевод из BIN в десятичную систему      | `1101 (bin)` → `13`        |
| `(cmd, n)`    | Применяет команду к `n` последним словам | `one two (up, 2)` → `ONE TWO` |

### 🔹 Обработка вложенных команд
- Поддержка сложных конструкций: `(Up (low))`, `(L(low)O(low)W(low))`
- Многопроходная обработка от внутренних команд к внешним

### 🔹 Обработка кавычек и сокращений
- Правильная расстановка пробелов вокруг кавычек
- Объединение сокращений: `it ' s` → `it's`

## 🚀 Установка и запуск

### 1️⃣ Клонируй репозиторий
```bash
git clone https://github.com/aaristan/go-reloaded.git
cd go-reloaded
```

### 2️⃣ Подготовь входной файл
Создай текстовый файл, например `sample.txt`:
```
harold wilson (cap, 2) : ' I am a optimist ,but a optimist who carries a raincoat . '
it (cap) was the best of times (up, 3) , it was the worst of times (low, 4) .
home(Up (low))
ABC (L(low)O(low)W(low))
```

### 3️⃣ Запусти программу
```bash
go run . sample.txt result.txt
```
- ✅ Первый аргумент — путь к **входному** файлу
- ✅ Второй аргумент — путь к **выходному** файлу

### 4️⃣ Посмотри результат
```bash
cat result.txt
```

**Вывод:**
```
Harold Wilson: 'I am an optimist, but an optimist who carries a raincoat.'
It was THE BEST OF times, it WAS THE WORST of times.
HOME
abc
```

## 📋 Примеры работы

| Входной текст | Результат |
|---------------|-----------|
| `it (cap) was a amazing (cap) experience ?!` | `It was an Amazing experience?!` |
| `A a apple an hat an university` | `A an apple a hat a university` |
| `Hello (up) world (low)` | `HELLO world` |
| `1101 (bin) 1F (hex)` | `13 31` |
| `one two three (cap, 2)` | `one Two Three` |
| `home(Up (low))` | `HOME` |
| `ABC (L(low)O(low)W(low))` | `abc` |

## ⚙️ Логика работы

1. **Токенизация** — разбивка текста на слова, знаки препинания и команды с помощью регулярных выражений
2. **Обработка вложенных команд** — многопроходная обработка сложных конструкций со скобками
3. **Применение команд** — последовательное выполнение `(up)`, `(cap)`, `(bin)`, `(hex)` и др.
4. **Коррекция артиклей** — интеллектуальное исправление a/an с учётом фонетики
5. **Реконструкция текста** — восстановление с правилами пунктуации, пробелов и кавычек

---

## ❗ Примечания

- Программа принимает **ровно два аргумента** — путь к входному и выходному файлу
- В случае неверного вызова программа выведет подсказку:
  ```
  Usage: go run . <input.txt> <output.txt>
  ```
- Все изменения применяются последовательно и атомарно
- Поддерживается обработка больших файлов благодаря потоковой обработке
- Код полностью самодостаточен и не требует внешних зависимостей
